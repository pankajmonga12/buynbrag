3. GET /index.php/rapiv1/ldata/get/<page_number>
returns: applicaton/json
returns a json object with a property named "result" whose value contains an array of objects in the following format:

				{
					"productID": "11700",
					"storeID": "186",
					"catID": "8",
					"productName": "Spin Up",
					"hasFancied": "0",
					"hasbragged": "0"
				}

				-- the productID contains the unique identifier of a product
				-- the storeID contains the unique identifier of the store to which the product belongs
				-- the productName contains the name of product
				-- the hasFancied property has a value of "0" or "1" specifying whether -- the current user has fancied the product(0) or not(1).
				-- the hasBragged property has a value of "0" or "1" specifying whether -- the current user has bragged the product(0) or not(1).

PLEASE NOTE that this API is used only when a user is not logged-in in which case they are presented with a selected set of products using this API

5. POST /index.php/async/login

expects the following n-v pair:

signin_email: the email address being used to sign-in
signin_pw: the password for the account specified by signin_email parameter

response returned (sample):

{
	"0":{
			"sessionSet":true,
			"validCredentials":true,
			"ID":"141",
			"gender":"male",
			"FBID":"707544199",
			"fullName":"Shammi Shailaj",
			"isActive":"1",
			"rFlowStatus":"6"
		},
	"1":{
			"rank":"11",
			"oldRank":"11",
			"tNewRank":18,
			"fancyCount":"514",
			"cartCount":0,
			"totalBadges":"41",
			"storeURL":"http:\/\/buynbragstores.s3.amazonaws.com\/",
			"baseURL":"https:\/\/buynbrag.com\/"
		},
	"2":{
			"userExists":true,
			"newUserCreated":false,
			"newUserID":null
		},
	"3":{
			"badgesData":{}
		}
}

sessionset - a value of true means the session was set and the user was logged-in successfully otherwise false
validCredentials - a  value of true means that the credentials provied to the API were found in the DB and matched successfully against them otherwise false
ID - unique user identification number
gender - male/ female
FBID - facebook ID of the user if they have any otherwise it contains a value of either null or "null" or ""
fullName - full name of the user
isActive - whether a user's account is active or not (1 = active, 0 = inactive)
rFlowstatus - should contain a value of 6 for a user to see the member's first page otherwise they are presented with the registration flow
rank - rank of the user
oldRank - rank of the user according to the old algorithm
tNewrank - rank of the user using the most recent algorithm
fancyCount - total products bragged by a user
cartCount - number of products in the users cart
totalBadges - total number of badges earned by a user
storeURL - the store URL which is used as the base URL while displaying images
baseURL - current base URL from which the API is being served
userExists - a value of true means the user exists in the DB otherwise false. This was being used earlier, of no use now
newUserCreated - a value of true means the account was created successfully otherwise false. This was being used earlier, of no use now
newUserID - user's unique identification number that was generated after creation of the account.This was being used earlier, of no use now


badgesData - Dated. Use async/badgeNotifications instead

									"discountAmount":"0",
									"availableQuantity":"9"
								}
						},
			"cartCount":1
		}
}

isLoggedIN - true if a user is logged-in, false otherwise

productsVariants - product variants if any, null otherwise

productID - unique identification of a product
cartID - unique identification of the product in the cart
storeID - unique identification number of the store to which the product belongs
quantity - number of pieces of the product being requested by the customer
vsize - size variant if any, of the product being requested by the customer
vcolor - colour variant if any of the product being requested by the customer
productName - name of the product
isOnDiscount - specifies whether a product is on discount(1) or not(0). If a product is on discount, then the effective price of that product becomes (originalPrice - discountAmoount)
processingTime - the number of business days required to process the order only after which the shipping is initiated by the seller
discountAmount - the amount to be discounted while displaying the final price if isOnDiscount has a value of 1(one)
availableQuantity -  the quantity of the product that is available in the stock
cartCount - the number of products in the cart
